### FRONTEND

#### React Patterns — 3/5

**Cosa va bene:**
- React Hook Form + zodResolver per validazione form
- `useCallback` per setter generico in ComposeEditor
- Cleanup corretto negli useEffect (cancelled flag, timer cleanup)

**Cosa migliorare:**
| Issue | File:linea | Skill | Impatto |
|-------|-----------|-------|---------|
| ComposeEditor e' un "god component" (1024 righe, 15+ useState) | `ComposeEditor.tsx:110-851` | react-patterns | Alto — decomposizione in custom hooks |
| Array index usato come key su liste riordinabili | `ComposeEditor.tsx:584,979,806` | react-patterns | Medio — stale state su reorder |
| emailHtml calcolato a ogni render senza useMemo | `ComposeEditor.tsx:295` | react-patterns | Basso — performance |
| SuccessMessage ha "use client" senza necessita' | `SuccessMessage.tsx:1` | react-patterns | Basso — bundle inutile |

**Quick wins:**
1. `useMemo` su `buildEmailHtml(templateData)`
2. ID stabili per eventi/artisti (`crypto.randomUUID()`) al posto di index come key
3. Rimuovere `"use client"` da SuccessMessage.tsx
